前端开发规范

2024-01-19
Uncategorized
3.3k words

俗话说“没有规矩，不成方圆”。自由是建立在规则之上的，没有了规则，就没有真正的自由。

本规范旨在减少团队协作成本和维护成本，让代码阅读起来更容易，使开发流程更加规范化。

环境配置

本地环境配置

node.js 版本：20.10.0

建议使用 vscode

eslint + prettier

  • 配置 prettier format on save

    1. 安装 Prettier

      1. 点击 VS Code 的扩展图标

      2. 搜索 prettier

      3. 在搜索结果中找到 Prettier

      4. 点击 install 按钮

      5. 安装完成后点击 Reload 按钮或者重启 VS Code

    2. 配置 Prettier 为默认代码格式化工具

      1. 同时按下 command + shift + p 键盘（Mac）

      2. 输入 format document with

      3. 选择 Configure Default Formatter…

      4. 选择 Prettier 为默认的工具

    3. 配置文件保存时自动格式化

      1. command + shift + p 打开命令窗口

      2. 输入 setting

      3. 选择 Preferences: Open User Settings 选项

      4. 搜索 format on save 配置项，选中复选框

    4. .prettierrc.cjs

  • 配置 eslint

    • 配置 eslint 插件

    • 配置内容详见.eslintrc.js

开发规范

目录构建规范

设计目的：与项目开发的目录结构保持一致，便于项目的构建与管理。

  • 资源分类

    • 源代码资源：指开发者编写的源代码，包括 html、css、template 等

    • 内容资源：指希望作为内容提供给访问者的资源，包括图片、字体、flash、pdf 等，最常用的应该就是 readme.md 文件。

  • 具体规范

    • 命名原则：简洁明了

      • egs: config -> 配置相关

    • 根据业务逻辑进行文件夹的划分

      • src 目录名词解释：

        • api: 用于统一设置各种后端请求接口

        • assets: 存放静态资源（图片）

        • common：存放配置、枚举值 & 其他常量

        • components：存放公共组件

        • directives：自定义指令

        • hooks：hook 函数

        • router：前端路由

        • setup：注册插件

        • store：使用 pinia API

        • styles：公共样式

        • utils：工具

          • axios

        • views：页面组件

CSS 命名规范

BEM 规范

BEM 代表 块(block)，元素(element)，修饰符(modifier)

解决问题：组件之间的完全解耦，不会造成命名空间的污染

  • 在选择器中，由以下三种符号来表示扩展的关系：

    • 中划线 ：仅作为连字符使用，表示某个块或者某个子元素的多单词之间的连接记号；
    • 双下划线：双下划线用来连接块和块的子元素；
    • 单下划线：单下划线用来描述一个块或者块的子元素的一种状态；
    • 示例：type-block**element_modifier

  • 一个块中元素的类名必须用父级块的名称作为前缀。

OOCSS 规范

Object Oriented CSS，面向对象的 CSS，旨在编写高可复用、低耦合和高扩展的 CSS 代码

  • 思考在不同元素中哪些样式是通用的，然后将这些通用的样式从模块、组件、对象等中抽离出来，使其能在任何地方能够复用，而不依赖于某个特定的容器。

产品设计

在开始写代码前，书写产品设计文档，可有效规避开发时出现的许多业务逻辑需求相关问题，提高效率，方便后期维护。

  • 背景

    • 为了解决什么问题，启动了这个项目。

  • 优先级

    • 序号越小优先级越高。

  • 技术选型

    • 项目采用什么技术，比如 Vue、react…，以及其他一些配套的插件、组件库、构建工具。

  • 接口说明

    • 采用什么数据传输协议、传输数据格式、数据结构等。

  • 使用说明

    • 需要什么样的传参（名称，类型，说明，默认值），输出什么样的结果。
