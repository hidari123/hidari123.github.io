俗话说“没有规矩，不成方圆”。自由是建立在规则之上的，没有了规则，就没有真正的自由。

本规范旨在减少团队协作成本和维护成本，让代码阅读起来更容易，使开发流程更加规范化。

## 开发规范

### 目录构建规范

> 设计目的：与项目开发的目录结构保持一致，便于项目的构建与管理。

- 资源分类

  - 源代码资源：指开发者编写的源代码，包括 html、css、template 等

  - 内容资源：指希望作为内容提供给访问者的资源，包括图片、字体、flash、pdf 等，最常用的应该就是 readme.md 文件。

- 具体规范

  - 命名原则：简洁明了

    - egs: config -> 配置相关

  - 根据业务逻辑进行文件夹的划分

    - src 目录名词解释：

      - api: 用于统一设置各种后端请求接口

      - assets: 存放静态资源（图片）

      - common：存放配置、枚举值 & 其他常量

      - components：存放公共组件

      - directives：自定义指令

      - hooks：hook 函数

      - router：前端路由

      - setup：注册插件

      - store：使用 pinia API

      - styles：公共样式

      - utils：工具

        - axios

      - views：页面组件

## 前端代码规范

### CSS 规范

- 尽量使用整十数/双数。

- 尽量使用 tailwindcss 简化样式。

- 尽量避免出现`!important`。

### 颜色规范

- 使用 tailwindcss 颜色设计 -> `tailwind.config.js`

**步骤**

1. 下载安装

```ssh
$ npm install tailwindcss
```

2. 配置

```js
// tailwind.config.js
module.exports = {
  theme: {
    colors: {
      // Configure your color palette here
    },
  },
};
```

3. 使用

   1.

```css
/* theme() */
.my-class {
  border: 1px solid theme("colors.primary");
}
```

2.

```css
.my-class {
  @apply text-primary;
  @apply bg-primary;
}
```

antd 修改主题颜色配置 -> `config\webpack.config.base.js`

### CSS 命名规范

#### BEM 规范

> BEM 代表 块(block)，元素(element)，修饰符(modifier)

> 解决问题：组件之间的完全解耦，不会造成命名空间的污染

- 在选择器中，由以下三种符号来表示扩展的关系：

  - 中划线 ：仅作为连字符使用，表示某个块或者某个子元素的多单词之间的连接记号；
  - 双下划线：双下划线用来连接块和块的子元素；
  - 单下划线：单下划线用来描述一个块或者块的子元素的一种状态；
  - 示例：`type-block**element_modifier`

- 一个块中元素的类名必须用父级块的名称作为前缀。

#### OOCSS 规范

> Object Oriented CSS，面向对象的 CSS，旨在编写高可复用、低耦合和高扩展的 CSS 代码

- 思考在不同元素中哪些样式是通用的，然后将这些通用的样式从模块、组件、对象等中抽离出来，使其能在任何地方能够复用，而不依赖于某个特定的容器。

## 代码书写规范

### JavaScript 编写规范

- 使用 eslint 规范

### 包引入规范

- 每种路径间空一行

- 引入顺序

  1. 第三方包

  2. 第二方包，绝对路径

  3. 相对路径

- 引入格式

  - 后缀名统一

### 页面渲染规范

`template`中过长的逻辑用`computed`书写，写上注释方便查看&修改。

尽量避免在渲染时出现计算。

### 文件命名规范

`hook`函数使用`use`开头。

## 产品设计

> 在开始写代码前，书写产品设计文档，可有效规避开发时出现的许多业务逻辑需求相关问题，提高效率，方便后期维护。

- 背景

  - 为了解决什么问题，启动了这个项目。

- 优先级

  - 序号越小优先级越高。

- 技术选型

  - 项目采用什么技术，比如 Vue、react...，以及其他一些配套的插件、组件库、构建工具。

- 接口说明

  - 采用什么数据传输协议、传输数据格式、数据结构等。

- 使用说明

  - 需要什么样的传参（名称，类型，说明，默认值），输出什么样的结果。
